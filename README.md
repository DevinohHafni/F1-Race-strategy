# 🏎️ F1 Race Optimization Analysis – Power BI Dashboard

## 📌 Project Overview
This project is an end-to-end **Formula 1 race analytics solution** designed to analyze driver performance, tire strategies, pit stop efficiency, and overall race optimization patterns.

The dataset is processed using Python and stored in PostgreSQL, followed by visualization in Power BI to deliver actionable insights similar to real-world motorsport performance analysis systems.

## 🎯 Objectives
- Analyze driver-wise lap time performance
- Evaluate impact of tire compounds on race performance
- Study pit stop behavior and efficiency
- Identify performance trends across teams and drivers
- Build an interactive dashboard for decision-making insights

---

## 📊 Key KPIs
- Total Laps: 5,000+
- Total Drivers: 20
- Fastest Lap: 75.65 sec
- Slowest Lap: 105.72 sec
- Average Lap Time: 87.91 sec

---

## 🛠️ Tech Stack
- Python (Pandas)
- PostgreSQL
- Power BI
- FastF1 Dataset

---

## 🔄 Data Pipeline
- Data extraction from F1 telemetry dataset
- Data cleaning and preprocessing in Python
- Handling missing and invalid values
- Feature engineering (lap time, stint, pit indicators)
- Data storage in PostgreSQL (`public.race`)
- Power BI integration and dashboard development

---

## 📈 Dashboard Features

### Driver Performance Analysis
- Average lap time comparison across drivers
- Driver ranking based on performance consistency

### Tire Strategy Analysis
- Lap time variation across compounds (Soft, Medium, Hard, Wet)
- Tire degradation trends across stints

### Pit Stop Analysis
- Pit stop frequency per driver
- Impact of fresh tires on performance

### Race Insights
- Lap time trends across race
- Team-wise performance comparison

---

## 💡 Key Insights
- Tire compounds significantly impact lap time performance
- Drivers with optimized pit strategies show better consistency
- Performance degradation increases with stint length
- Mid-field teams show higher variability in lap times
